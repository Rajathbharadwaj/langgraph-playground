What is LangGraph?

LangGraph is a library for building stateful, multi-actor applications with LLMs. Think of it as a state machine framework specifically designed for AI agents.

Unlike basic LLM chains, LangGraph gives you:

  • Explicit state management - Know exactly what your agent "remembers"
  • Conditional routing - Branch based on LLM output or external conditions
  • Cycles and loops - Agents that can iterate and self-correct
  • Human-in-the-loop - Pause execution for human approval
  • Persistence - Save and resume agent state

Why LangGraph Over Plain LangChain?

LangChain is great for simple chains: prompt → LLM → output. But agents need more:

# Plain LangChain: Linear
Input → LLM → Tool → LLM → Output

# LangGraph: Stateful Graph
       ┌──────────────────┐
       │                  │
Input → LLM → Decision → Tool
       ↑         │         │
       └─────────┴─────────┘

The key difference: LangGraph agents can loop, branch, and remember. This is essential for production agents that need to handle errors, retry operations, and maintain context across multiple turns.

Core Concepts

1. State

State is a TypedDict that holds everything your agent needs to remember:

from typing import TypedDict, Annotated
from langgraph.graph import add_messages

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]
    current_task: str
    tools_called: list[str]
    iteration_count: int

2. Nodes

Nodes are functions that transform state. Each node receives the current state and returns updates:

def llm_node(state: AgentState) -> dict:
    response = llm.invoke(state["messages"])
    return {"messages": [response]}

def tool_node(state: AgentState) -> dict:
    # Execute tool based on last message
    result = execute_tool(state["messages"][-1])
    return {"messages": [result], "tools_called": [tool_name]}

3. Edges

Edges define how nodes connect. Conditional edges route based on state:

def should_continue(state: AgentState) -> str:
    last_message = state["messages"][-1]
    if last_message.tool_calls:
        return "tools"  # Go to tool node
    return "end"  # Finish

graph.add_conditional_edges(
    "llm",
    should_continue,
    {"tools": "tool_node", "end": END}
)

Building Your First Agent

Let's build a simple ReAct agent that can answer questions and use tools:

from langgraph.graph import StateGraph, END
from langchain_anthropic import ChatAnthropic

# 1. Define state
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]

# 2. Create LLM with tools
llm = ChatAnthropic(model="claude-sonnet-4-20250514")
llm_with_tools = llm.bind_tools([search_tool, calculator_tool])

# 3. Define nodes
def call_model(state):
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

def call_tools(state):
    last_message = state["messages"][-1]
    results = []
    for tool_call in last_message.tool_calls:
        result = execute_tool(tool_call)
        results.append(ToolMessage(content=result, tool_call_id=tool_call["id"]))
    return {"messages": results}

# 4. Build graph
graph = StateGraph(AgentState)
graph.add_node("agent", call_model)
graph.add_node("tools", call_tools)

graph.set_entry_point("agent")
graph.add_conditional_edges("agent", should_continue, {"tools": "tools", "end": END})
graph.add_edge("tools", "agent")  # After tools, go back to agent

# 5. Compile and run
app = graph.compile()
result = app.invoke({"messages": [HumanMessage(content="What's 25 * 4?")]})

Human-in-the-Loop Patterns

Production agents often need human approval before taking actions. LangGraph makes this easy with checkpoints:

from langgraph.checkpoint.memory import MemorySaver

# Add checkpointing
memory = MemorySaver()
app = graph.compile(checkpointer=memory, interrupt_before=["dangerous_action"])

# Run until interrupt point
config = {"configurable": {"thread_id": "user_123"}}
result = app.invoke(inputs, config)

# Later: Get human approval and continue
# Human reviews the state...
if human_approves:
    result = app.invoke(None, config)  # Continue from checkpoint

Production Deployment

Deploying LangGraph agents to production requires:

  • Persistent checkpoints - Use PostgreSQL or Redis instead of memory
  • Error handling - Wrap nodes in try/except, implement retries
  • Observability - Add logging at each node transition
  • Rate limiting - Protect against runaway agents
  • Timeouts - Set maximum execution time
# Production-ready graph
from langgraph.checkpoint.postgres import PostgresSaver

checkpointer = PostgresSaver.from_conn_string(DATABASE_URL)
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["send_email", "execute_trade"],  # Human approval points
)
